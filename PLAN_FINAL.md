아래는 `PLAN.md`에 있었지만 PDF 벤치마크에 맞게 표현을 바꿔야 하는 항목이다.

## 8. UX 톤 메모

사이트 말투를 억지로 귀엽게 만들 필요는 없다. 대신 사용자 경험 곳곳에 `작게 웃기는 장치`와 `12기 내부 공유용 센스`가 들어가면 좋다.

방향은 아래 정도로 잡는다.

- 정답/오답 피드백은 건조하지 않게 짧은 위트 추가  
  예: `정답. 이건 강사님이 거의 찝어줬다.` / `아깝다. 이건 함정이 꽤 노골적이었다.`
- 계산형 정답 시 작은 성취감 연출  
  예: `숫자 감각 살아있다.` / `밸류에이션 감각 합격.`
- 족집게 문항은 시각적으로 따로 표시  
  예: `강사님 강조`, `언급된 문제`, `함정주의`
- 과한 캐릭터성 대신 시험 맥락에 맞는 신선한 카피 사용
- 전체 비주얼도 `귀엽다`보다 `센스 있다`, `조금 웃기다`, `의외로 공들였다`는 느낌을 목표로 함

이 부분은 실제 UX 상세 규칙을 잡을 때 더 구체화하면 된다.
